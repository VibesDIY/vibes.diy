# Draining a saturated Dependabot queue with one bundled PR (and teaching the skill to do it)

Source: grouped low/mid-risk Dependabot bump, branch `claude/dependabot-review-skill-wngcux`. The `dependabot-review` audit found the queue at **20/20** — `open-pull-requests-limit: 20` means Dependabot was silently blocked from opening *any* new PR, including security ones, until merges happened. The fix isn't "merge 20 PRs one at a time"; it's roll the safe tier into a single PR so they share one lockfile resolution and one validation pass.

The hook: **"safe to merge" and "safe to merge *separately*" are different claims.** Twelve PRs each touching `pnpm-lock.yaml` is twelve racing resolutions and twelve `pnpm dedupe --check` gates; bundle them and it's one. The bundle was the `Merge` + `Verify` verdicts (libsql, isbot, react-hook-form, typescript-eslint, playwright, shiki+@shikijs/monaco, the grouped @clerk/*, esbuild, knip, eslint, @typescript/native-preview). The `Hold`/`Investigate` ones — React Router 8 (removes `react-router-dom`, 18 import sites), `cookie` 2.0 (renamed exports break 3 server call sites), ag-grid 36 — stayed out because they need code, not a version flip.

The gotcha worth preserving: **the bundle is exactly where a long-deferred risky bump tries to sneak through.** A `multiformats` 13→14 major had been open 39 days while everything else was 1–12 days old. Age relative to the batch is signal — if it survived a prior weekly cycle, something passed it over for a reason that isn't in the diff. So the rule baked into the skill (new Step 6 + an "Age vs. the batch" risk-table row): a Merge/Verify verdict gets it *considered*, but a stale-skipped item is surfaced as its own line, never silently folded into the group. Draining the queue must not become the mechanism that finally merges the thing everyone kept avoiding.

The other non-obvious bit: **bumping a floor in one workspace isn't enough.** Same package lives in many `package.json`s (isbot in 3, @libsql/client in 3, eslint in 3), and `pnpm dedupe --check` — which CI enforces — fails if you miss one. The faithful move is `grep -rn '"<pkg>":' --include=package.json` for every declaration site, bump them all, then `pnpm install && pnpm dedupe`. And note the difference between "what the PR title says" and "what package.json declares": eslint's title read `10.4.0 → 10.6.0` but the declared range was `^10.0.3` (10.4.0 was just the old *resolved* version) — Dependabot raises the floor to `^10.6.0`, so that's what the bundle does too.

Last: on merge, **close the originals by hand.** The closing commits live on the group branch, not Dependabot's, so GitHub won't auto-close them — leave that step out and the queue stays falsely at capacity.
