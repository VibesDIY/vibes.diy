# SSE Fixtures

Captured raw SSE responses for testing the v2 streaming pipeline.

## Approach

Schema is sent as a **system message** asking the model to return a JSON code block.
This mirrors the production path in `vibes.diy/vibe/srv-sandbox/srv-sandbox.ts` (`vibeCallAI` handler).

No `response_format` or `tool_use` — the model is simply prompted to return JSON.

The system message is generated by `build-schema-prompt.ts` (shared with srv-sandbox)
so fixtures always match the production prompt shape.

## Providers

| Fixture | Provider | Notes |
|---------|----------|-------|
| `openrouter-gpt.llm.txt` | OpenRouter → GPT-4o-mini | OpenRouter SSE format |
| `openrouter-claude.llm.txt` | OpenRouter → Claude Sonnet | OpenRouter SSE format |
| `openai.llm.txt` | OpenAI direct | OpenAI SSE format |
| `anthropic.llm.txt` | Anthropic direct | Anthropic SSE format (`text_delta`) |

All fixtures request a sandwich JSON: `{ "name": "...", "layers": ["..."] }`

## Anthropic Direct

Anthropic's native SSE format uses `event:` prefix lines and a different payload structure
(`content_block_delta`) that `SseChunk` does not yet handle. The fixture captures what a
future Anthropic SSE normalizer must consume. Captured via `curl` because Anthropic
requires `x-api-key` auth (not `Authorization: Bearer`).

## Regenerating fixtures

```bash
cd call-ai/v2
bash fixtures/capture.sh
```

Requires `.env` with: `OPENROUTER_API_KEY`, `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`

## Files

- `sandwich-schema.json` — schema used in the capture prompt
- `print-schema-prompt.ts` — prints the system message (used by capture.sh)
- `../build-schema-prompt.ts` — shared function that builds the system message
- `capture.sh` — shell script to re-capture all fixtures
